Chapter 02

卡住企业 Agent 自动化的四个真问题

在实际推进过程中，我发现阻碍企业 Agent 真正替代人工的，
不是某个单点的技术问题，而是一组系统性的难题：

1. 上下文拼不齐

信息分散，AI 拿不全

做一个项目立项决策，需要的信息可能涉及：战略方向、行业趋势、历史项目数据、团队资源情况、客户反馈……这些信息分散在不同系统、不同人手里。AI 拿不全信息，就做不出靠谱的判断。

💡 我的动作：绘制决策链路地图，识别关键节点

2. 多系统之间不互通

系统各自为政，数据孤岛

一个管理动作可能横跨项目管理系统、审批系统、文档系统、沟通工具、数据库。系统之间各自为政，Agent 要完成一个完整任务，必须能跨系统调取和写入数据。

💡 我的动作：梳理系统数据接口，建立跨系统数据映射表

3. 权限极其复杂

任务级权限编排难题

企业里没有两个人拥有完全相同的信息访问范围。一旦 Agent 要自动处理任务，权限问题马上变得非常棘手：它有时候要继承用户的权限，有时候要被降权，有时候又需要为了完成任务拿到临时授权。这本质上不是"开或关"的权限问题，而是"任务级别的权限编排"问题。

💡 我的动作：设计任务级权限编排模型，引入权限审计日志和权限沙盒机制

4. 技术路线变化快

技术迭代快，企业变更成本高

过去一年多，Agent 落地的主流技术方向一直在切——从 RAG 到工作流编排、再到多 Agent 协作、再到 MCP。技术迭代是快的、低成本的，但企业换方案涉及的是已建好的数据底座、安全审查、合规流程、实施团队和人员培训的整体变更，代价巨大。

💡 我的动作：AI 系统和业务系统保持松耦合架构，AI 层可快速替换
