## 1. 项目概述

### 1.1 项目背景

用户需要将一份 AI 项目管理实战经验文档打造成前端面试作品，面试岗位是**AI 产品经理**。目标是让面试官通过网页作品在 **3-5 分钟内** 快速建立清晰的人才画像，客观评估与招聘岗位的匹配度。

### 1.2 核心价值主张

- **不是简单的文档展示**，而是通过视觉化、交互式设计让信息更易消费
- **展示问题洞察力**——能看到别人看不到的真问题
- **展示方法论沉淀**——有框架、有体系，不是零散经验
- **展示实战结果**——有数据、有产出、可验证
- **展示可迁移能力**——这套方法能复用到新公司

### 1.3 目标用户

| 用户类型 | 需求 | 使用场景 |
|----------|------|----------|
| 面试官（HR） | 快速了解候选人背景 | 筛选简历后进一步评估 |
| 面试官（业务负责人） | 评估候选人能力匹配度 | 面试前准备/面试中参考 |
| 面试官（高管） | 评估候选人思维深度 | 终面环节 |

## 6. 交互设计规范

### 6.1 动画原则

| 原则 | 说明 |
|------|------|
| 有意义 | 动画服务于内容理解，不是装饰 |
| 适度 | 不过度使用动画，避免干扰阅读 |
| 流畅 | 所有动画 ≥60fps |
| 一致 | 相同类型的动画行为一致 |

### 6.2 核心动画

| 动画类型 | 触发条件 | 参数 |
|----------|----------|------|
| 淡入 + 上滑 | 滚动进入视口 | duration: 600ms, ease-out |
| 卡片展开 | 点击 | duration: 400ms, ease-out |
| 数字滚动 | 滚动进入视口 | duration: 1500ms |
| 图表绘制 | 滚动进入视口 | duration: 2000ms |
| 飞轮旋转 | 持续 | 10s/圈，线性 |

### 6.3 Hover 效果

| 元素 | Hover 效果 |
|------|------------|
| 卡片 | 上浮 4px + 阴影加深 |
| 按钮 | 上浮 2px + 背景变深 |
| 链接 | 下划线 + 颜色变深 |
| 数据卡片 | 上浮 + 边框高亮 |

## 7. 工作流程

### 7.1 设计师工作流程

```
1. 阅读 PRD 概览（本文档）
   ↓
2. 按模块顺序阅读产品文档
   ↓
3. 输出线框图（Wireframe）
   ↓
4. 输出高保真设计稿（Figma）
   ↓
5. 标注切图（自动/手动）
   ↓
6. 交付开发者
```

### 7.2 开发者工作流程

```
1. 阅读 PRD 概览
   ↓
2. 搭建项目脚手架
   ↓
3. 按模块顺序开发（01→08）
   ↓
4. 自测功能 + 响应式
   ↓
5. 联调整体效果
   ↓
6. 性能优化
   ↓
7. 部署上线
```

### 7.3 推荐开发顺序

| 阶段 | 模块 | 预计工时 |
|------|------|----------|
| Phase 1 | 项目 scaffold、配置、基础组件 | 4h |
| Phase 2 | Hero Section + Navigation | 4h |
| Phase 3 | Chapter 1-2（问题洞察 + 四个真问题） | 6h |
| Phase 4 | Chapter 3（实战解法·核心） | 8h |
| Phase 5 | Chapter 4（结果数据） | 6h |
| Phase 6 | Chapter 5-6（认知沉淀 + 能力映射） | 6h |
| Phase 7 | Chapter 7 + Footer | 4h |
| Phase 8 | 联调、优化、部署 | 6h |
| **总计** | | **44h** |
